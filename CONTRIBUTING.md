# Contributing to GOTT Protocol

Terima kasih atas ketertarikan lu di GOTT! Kami terbuka untuk kontribusi dari komunitas.

## Ways to Contribute

### 🐛 Report Bugs
- Gunakan GitHub Issues
- Sertakan langkah reproduksi yang jelas
- Sertakan environment info (browser, wallet, network)

### 💡 Suggest Features
- Buat Issue dengan label `enhancement`
- Jelaskan use case
- Diskusikan di Discord/Telegram dulu untuk ide besar

### 📝 Improve Documentation
- Typo fix → langsung PR
- Content update → diskusikan di Issue dulu
- Translation → sangat welcome (bahasa Indonesia prioritas)

### 💻 Contribute Code
- Fork repo
- Create feature branch (`git checkout -b feature/amazing-feature`)
- Commit dengan convention (`feat:`, `fix:`, `docs:`, etc)
- Push ke branch (`git push origin feature/amazing-feature`)
- Open Pull Request

### 🌐 Translate
- Bahasa Indonesia (primary, utama)
- English
- Bahasa Vietnam, Tagalog, Thai (Phase 2)

## Code Style

### Solidity
- Follow OpenZeppelin style guide
- NatSpec comments untuk public functions
- Custom errors instead of require strings (gas optimization)

### JavaScript/TypeScript
- ESLint + Prettier config provided
- TypeScript strict mode
- Functional style preferred

### Commit Messages
Follow [Conventional Commits](https://www.conventionalcommits.org/):
- `feat:` New feature
- `fix:` Bug fix
- `docs:` Documentation only
- `style:` Formatting, whitespace
- `refactor:` Code restructure, no feature change
- `test:` Add/update tests
- `chore:` Maintenance, dependencies

## Pull Request Process

1. Update CHANGELOG.md dengan entry baru
2. Update documentation jika perlu
3. Pastikan tests pass (saat tests ada)
4. Request review dari maintainer
5. Maintainer approve + merge
